# Бот для изучения слов — инструкция по запуску

## Что внутри
- `bot.py` — основной код бота
- `database.py` — работа с базой данных (SQLite, создаётся автоматически)
- `config.py` — настройки (токен, ID учителей)
- `requirements.txt` — список библиотек

## Команды бота
- `/start` — регистрация
- `/learn слово - перевод` — ученик добавляет своё слово
- `/catalog` — список слов от учителя, можно добавить себе нажатием кнопки
- `/train` — тренировка (бот сам решает, какие слова пора повторить)
- `/stats` — статистика по словам
- `/addwords` — (только для учителя) загрузка списка слов для всех учеников

## Шаг 1. Узнать свой Telegram ID (для учителя)
Напиши в Telegram боту **@userinfobot** — он пришлёт твой числовой ID. Он понадобится в шаге 3.

## Шаг 2. Создать аккаунт на Render
1. Зайди на https://render.com и зарегистрируйся (можно через GitHub).
2. Тебе нужно будет загрузить эти файлы в GitHub-репозиторий — самый простой способ:
   - Зарегистрируйся на https://github.com (если ещё нет аккаунта).
   - Создай новый репозиторий (кнопка "New repository"), например `wordbot`.
   - На странице репозитория нажми "Add file" → "Upload files" и загрузи туда все файлы из этой папки (`bot.py`, `database.py`, `config.py`, `requirements.txt`).

## Шаг 3. Создать сервис на Render
Важно: бесплатный тариф Background Worker на Render сейчас недоступен, поэтому используем **Web Service** (он бесплатный) — код бота уже подготовлен так, чтобы это работало корректно.

1. В Render нажми "New" → **Web Service**.
2. Подключи свой GitHub-репозиторий с ботом.
3. В настройках укажи:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `python bot.py`
4. Выбери тариф **Free**.
5. В разделе **Environment Variables** добавь:
   - `BOT_TOKEN` — токен, который тебе дал @BotFather
   - `ADMIN_IDS` — твой Telegram ID из шага 1 (если учителей несколько — через запятую: `111,222`)
6. Нажми "Create Web Service". Render сам установит зависимости и запустит бота.

Примечание: на бесплатном Web Service Render может "усыплять" сервис после 15 минут без веб-запросов и будить с задержкой ~30-60 секунд при следующем обращении. Для телеграм-бота это означает, что после долгого простоя первое сообщение ученика может прийти с небольшой задержкой — дальше бот будет работать как обычно. Если это станет проблемой, можно подключить бесплатный сервис типа UptimeRobot, который будет "пинговать" бота каждые несколько минут, чтобы он не засыпал — подскажи, если понадобится, помогу настроить.

## Шаг 4. Проверка
Через пару минут зайди в своего бота в Telegram и напиши `/start`. Если ответил — всё работает.

Дальше как учитель: напиши `/addwords` и пришли список слов построчно вида:
```
apple - яблоко
cat - кошка
table - стол
```
Ученики смогут увидеть их через `/catalog` и добавить себе, либо добавлять свои через `/learn слово - перевод`.

## Важно про базу данных на Render
Бесплатный план Render пересоздаёт файловую систему при каждом передеплое — это значит, что база `wordbot.db` (а с ней прогресс учеников) может слететь, если ты будешь часто обновлять код. Для теста это не страшно. Если бот приживётся и начнёт активно использоваться, через пару недель имеет смысл подключить постоянное хранилище (Render Disk, это тоже недорого) или перенести базу в бесплатный облачный Postgres — напиши мне, когда дойдёт до этого, помогу перенести.

Скопируй текст ошибки из вкладки "Logs" на Render и пришли мне — разберёмся.
